# タスク一覧

## フェーズ説明
- フェーズ1: プロトタイプ（コア戦闘ループ、最小システムの自動化確認）
- フェーズ2: コンテンツ拡張（武器/敵/障害物/AIの網羅）
- フェーズ3: バランス/UX確定（数値調整、UI/ログ/死因レポート整備）
- フェーズ4: 最終化（最終調整、バグ修正、配布準備）

## フェーズ2: コンテンツ拡張
### P2-1: 武器/敵/障害物/AIの拡張
- タイトル: 武器/敵/障害物/AIの追加
- 背景: プレイバリエーションを生む要素の網羅が必要。
- 詳細: 代表的な武器、敵種別、障害物、簡易AI行動パターンを追加する。
- 実装イメージ: データ駆動で定義を追加し、戦闘ループに反映。
- 確認観点: 種別ごとの挙動差が確認できること。
- 補足: バランスは後工程で調整。

### P2-2: Asset準備（スプライト/SE/BGM/UI）
- タイトル: Asset準備（スプライト/SE/BGM/UI）
- 背景: 演出とプレイ感の検証に必要。
- 詳細: 仮/本番候補のスプライト、SE、BGM、UI素材を整理し適用する。
- 実装イメージ: assets/ に素材を配置し、読み込み経路を整備。
- 確認観点: 各素材がゲーム内で参照されること。
- 補足: 著作権とライセンスを確認する。

## フェーズ3: バランス/UX確定
### P3-1: 数値バランス調整
- タイトル: 数値バランス調整
- 背景: 適切な難易度とリテンションを確保するため。
- 詳細: HP/ダメージ/ドロップ/生成レートの調整と検証を行う。
- 実装イメージ: 設定値をパラメータ化し、テストプレイで調整。
- 確認観点: 目標プレイ時間と勝率が達成されること。
- 補足: 調整ログを残す。

### P3-2: UI/ログ/死因レポート整備
- タイトル: UI/ログ/死因レポート整備
- 背景: 改善サイクルを回すための観測性が必要。
- 詳細: UI情報提示、ログ整備、死因レポートの出力を追加する。
- 実装イメージ: UIに状態表示を追加し、ログに死因タグを付与。
- 確認観点: プレイ後に死因が判別できること。
- 補足: JSONログ形式を検討する。

## フェーズ4: 最終化
### P4-1: 最終調整とバグ修正
- タイトル: 最終調整とバグ修正
- 背景: リリース品質を担保する。
- 詳細: 既知不具合の修正と最終調整を行う。
- 実装イメージ: issue 化された内容を優先度順に対応。
- 確認観点: 重大バグが解消されていること。
- 補足: リリースブロッカーを明示。

### P4-2: 配布準備
- タイトル: 配布準備
- 背景: 配布物の生成と案内が必要。
- 詳細: ビルド/配布手順の整備、成果物の生成を行う。
- 実装イメージ: runbook/ に手順を記載し、ビルド成果物を作成。
- 確認観点: 手順通りに配布物が生成できること。
- 補足: バージョン付与ルールを確認。
