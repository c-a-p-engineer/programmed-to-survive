# アーキテクチャ設計（プロトタイプ方針）

## 目的
- デバッグしやすい構成で、短サイクルに改善できるプロトタイプを維持する。

## 設計方針
1. 疎結合
   - 設定値・ロードアウト・UI文言は `config.js` に集約し、ゲームロジックは `combat.js` に限定する。
   - 起動処理は `main.js` に集約し、ゲームロジックと依存を分離する。

2. データ/設定値の外出し
   - 機体/武器/AI/バランス/障害物配置などの値は `PTS_CONFIG` として外部化する。
   - ロジック側は `PTS_CONFIG` を参照するだけにし、パラメータ調整で動作を変えられる構成にする。

3. 責任分解とファイル分割
   - `config.js`: 定数・設定・バランス値の管理
   - `combat.js`: 戦闘ループとシーン管理
   - `main.js`: Phaser の起動とシーン登録

## 運用メモ
- 数値調整は `config.js` のみ変更すればよいように保つ。
- UI文言変更は `config.js` の `ui` セクションで完結させる。
